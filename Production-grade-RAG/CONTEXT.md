# Domain Context

This repository implements retrieval-augmented generation for Foundry Nuke documentation.

## Vocabulary

- Nuke page: A stored documentation page for a Nuke node or reference section. Pages live in PostgreSQL and retain raw content plus optional parsed sections.
- Chunk: A search-sized segment derived from a Nuke page. Chunks carry page metadata, text, an embedding, and a deterministic chunk identifier.
- Search backend: The configured chunk retrieval implementation. Supported values are `postgres_embedding` and `opensearch`.
- RAG answer: A response generated by retrieving chunks, constructing a prompt, calling the LLM provider, and returning cited source URLs.
- Agentic workflow: The LangGraph-based retrieval and answering path behind `/api/v1/ask-agentic`.
- Ingestion run: A scrape, save, chunk, embed, index, and reporting cycle driven by an orchestrator adapter.
- Knowledge graph extraction: Optional extraction of triples from indexed Nuke pages into Neo4j.

## Compatibility Rule

HTTP routes, request and response schemas, environment variable names, Docker ports, Compose profiles, SQL tables, and UI tool names are public compatibility surfaces. Internal modules can move only behind stable shims until all callers use the new module homes.
